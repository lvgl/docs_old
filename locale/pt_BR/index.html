Bem vindo a Biblioteca de interface grfica LittlevGL

LittlevGL um software grfico aberto com bibliotecas que providencia tudo que voc precisa para criar um GUI (Interface Grfica de Usurio), com elementoos fceis de usar e efeitos visuais bonitos e usando pouca memria

Principais caractersticas

  • Poderosos blocos de construo botes, grficos, listas, controles deslizantes, imagens, etc.
  • Grficos avanados com animaes, anti-aliasing, opacidade, deslizamento suave
  • Vrios dispositivos de entrada touch pad, mouse, teclado, codificadores, botes, etc.
  • Suporte multi-idioma com codificao UTF-8
  • Elementos grficos totalmente customizveis
  • Hardware independente para usar com qualquer microcontrolador ou display
  • Escalvel para operar com pequena quantidade de memria (50 KB de Flash, 10 KB de RAM)
  • OS, GPU e memria externa supportados (opcional)
  • nico frame buffer operaes at mesmo com avano de efeitos grficos
  • Escrito em C para mxima compatibilidade
  • Simulador para desenvolver no PC sem necessidade de um hardware especfico
  • Tutoriais, examplos, temas para desenvolvimento rpido

Escrito para v5.2

Arquitetura do sistema

Aplicao

Sua aplicao na qual cria o GUI e controlador (handle) para a tarefa especfica.

LittlevGL

Os prprios grficos da biblioteca. Sua aplicao pode comunicar com a biblioteca para criar a GUI. Ele contm uma interface HAL (Camada Abstrata de Hardware) para registrar seu display e drivers de entrada de dispositivos.

Driver

Junto ao seu driver especfico, a biblioteca contm funes para controlar seu display, opcionalmente uma GPU e leitura do touchpad ou botes.

Existem duas configuraes tpicas para hardware dependendo da MCU que pussui um driver perifrico LCD/TFT ou no. Em ambos os casoos, um frame buffer ser necessrio para armazenar a imagem atual na tela.

MCU com driver TFT/LCD

Se sua MCU tem um driver TFT/LCD ento voc pode conectar um display diretamente via interface RGB. Neste caso, o frame buffer pode estar na sia RAM interna (se o MCU possui RAM suficiente) ou na RAM externa (se a MCU possui uma interface de memria).

Controlador de display externo

Se o MCU no possui um driver TFT/LCD ento contrlador de display externo (I.e. SSD1963, SSD1306, ILI9341) precisa ser usado. Neste caso, o MCU pode se comunicar com o contralador do display via porta paralela, SPI ou algumas vezes I2C. O frame buffer usualmente localizado no controlador de display na qual economiza muita RAM para a MCU.

Requerimentos

  • Microcontrolador ou processador de 16, 32 ou 64 bit
  • Velocidade de clock de 16 MHz
  • 8 KB de RAM para dados estticos e >2 KB de RAM para dados dinmicos (objetos grficos)
  • 64 KB de memria permanente (flash)
  • Opcionalmente ~1/10 do tamanho da tela de memria para buffer interno (em 240 x 320, com cores de 16 bit de aprox. 15 KB)

O LittlevGL projetado para ser altamente portavel e no necessitar de nenhum recurso externo:

  • Nenhuma RAM externa requerida (mas suportado)
  • Nenhum nmero de vrgula flutuante so usados
  • Nenhuma GPU necessria (mas suportado)
  • Somente um nico frame buffer requerido e localizado em:
    • RAM interna ou
    • RAM externa ou
    • Controlador da memria do display externo

Se voc quiser reduzir o recurso de hardware requerido voc pode:

  • Desativar tipos de objetos no usados para economizar RAM e ROM
  • Mudar o tamanho do buffer grfico para economizar RAM
  • Usar estilos mais simples para reduzir o tempo de renderizao

Preparao do projeto

Obtendo a biblioteca

O Littlev Graphics Library est disponvel no GitHub: https://github.com/littlevgl/lvgl. Voc pode clonar ou fazer download da ltima verso da biblioteca daqui ou voc pode usar a pgina de Download tambm.

As bibliotecas grficas est no diretrio lvgl na qual deve ser copiado dentro do seu projeto.

Arquivo de configurao

Existe um arquivo cabealho de configurao para o LittlevGL: lv_conf.h. Ele configura o comportamento bsico da biblioteca durante o tempo de compilao. Desative os mdulos no usados, configuraes e ajuste do tamanho da memria de buffers, etc.

Copie _lvgl/lv_conf_templ.h_ prximo ao diretrio lvgl e renomeie-o para _lv_conf.h_. Abra o arquivo e apague o primeiro #if e o ltimo #endif para ativar o contedo. No arquivo de configurao, comentrios explicam o siginificado das opes. Verifique pelo menos essas trs opes e modifique-as de acordo com seu hardware.

  1. LV_HOR_RES Resoluo horizontal de seu display
  2. LV_VER_RES Resoluo vertical de seu display
  3. LV_COLOR_DEPTH 8 para (RG332), 16 para (RGB565) ou 32 para (RGB888 and ARGB8888).

Inicializao

Para usar a biblioteca voc tem que inicializ-la alm de outros componentes tambm. A ordem da inicializao :

  1. Chamar _lv_init()_
  2. Inicializar seus drivers
  3. Registrar o display e a entrada de dispositivos dos drives dentro do LittlevGL. (Veja abaixo)

Portando a biblioteca

Para colocar o LittlevGL dentro do seu projeto, primeiramente voc tem que providenciar algumas funes e registr-la na biblioteca de grficos.

Interface do display

Para configurar um display uma varivel do tipo lv_disp_drv_t deve ser inicializada:

lv_disp_drv_t disp_drv;
lv_disp_drv_init(&disp_drv); /*Inicializao bsica*/
disp_drv. ...=...; /*Inicializar os campos aqui. Veja abaixo.*/
lv_disp_drv_register(&disp_drv); /*Registrar o driver dentro do LittlevGL*/

Voc pode configurar o driver para diferentes modos de operao. Para aprender mais sobre modos de desenho, visite Desenhando e renderizando.

Bufferizao interna (VDB)

A biblioteca de grficos funciona com mecanismo de buffer para criar caractersticas de avanos grficos com somente um frame buffer. Um buffer interno chamado de VDB (Display Virtual de Buffer) a seu tamanho pode ser ajustado dentro do lv_conf.h com _LV_VDB_SIZE_. quando _LV_VDB_SIZE_ > 0 ento a bufferizao interna usada e voc tem que providenciar uma funo na qual esvazie o contedo desse buffer para o seu display:

disp_drv.disp_flush = my_disp_flush;
...
void my_disp_flush(int32_t x1, int32_t y1, int32_t x2, int32_t y2, const lv_color_t* color_p)
{
    /*TODO Copie 'color_p' para a rea especfica*/
    /*Chame 'lv_flush_ready()' quando estiver pronto*/lv_flush_ready();

Na funo flush, voc pode usar DMA ou qualquer hardware para fazer o despejo para o background, mas quando o despejo estiver pronto voc tem que chamar

lv_flush_ready();

Interface de dispotitivo de entrada

Para configurar um dispositivo de entrada uma varivel lv_indev_drv_t precisa ser inicializada:

lv_indev_drv_t indev_drv;lv_indev_drv_init(&indev_drv);  /*Inicializao bsica*/
indev_drv.type =.../*Veja abaixo.*/
indev_drv.read =.../*Veja abaixo.*/
lv_indev_drv_register(&indev_drv);  /*Registra o driver no LittlevGL*/

type pode ser

  • LV_INDEV_TYPE_POINTER: touchpad ou mouse
  • LV_INDEV_TYPE_KEYPAD: teclado
  • LV_INDEV_TYPE_ENCODER: esquerda, direita, apertar
  • LV_INDEV_TYPE_BUTTON: boto externo pressionando a tela

read um funo ponteiro na qual ser chamada periodicamente para reportar o estado corrente de um dispositivo de entrada. Ele pode tambm dar um buffer de dados e retornar false quando nenhum dados podem ser lidos ou true quando o buffer no est vazio.

To learn more about input devices visit Input devices. Para aprender mais sobre os dispositivos de entrada, visite Dispositivo de entrada.

Touchpad, mouse ou qualquer ponteiro

indev_drv.type = LV_INDEV_TYPE_POINTER;
indev_drv.read = my_input_read;

A leitura da funo deve ser algo parecido como:

bool my_input_read(lv_indev_data_t*data)
{
    data->point.x = touchpad_x;   
    data->point.y = touchpad_y;
    data->state = LV_INDEV_STATE_PR or LV_INDEV_STATE_REL;
    return false; /*Sem mais bufferizao, sendo assim, sem mais dados para ler*/
}

NOTA IMPORTANTE: Drivers de touchpad devem retornar os ltimas coordenadas X/Y at mesmo quando o estado LV_INDEV_STATE_REL.

Keypad ou teclado

indev_drv.type = LV_INDEV_TYPE_KEYPAD;
indev_drv.read = my_input_read;

A leitura da funo deve ser algo parecido como:

bool keyboard_read(lv_indev_data_t*data){
  data->key = last_key();            /*Obtm o ltima tecla pressionada ou liberada*/
  if(key_pressed()){
    data->state = LV_INDEV_STATE_PR;
  }
  else{
    data->state = LV_INDEV_STATE_REL;
  }
  return false; /*Sem mais bufferizao, sendo assim, sem mais dados para ler*/
}

Para usar um teclado:

  • Registre uma funo read (como acima) com o tipo _LV_INDEV_TYPE_KEYPAD_.
  • O _USE_LV_GROUP_ tem que estar ativado dentro do _lv_conf.h_
  • Um grupo de objeto tem que ser criado: _lv_group_create()_ e objetos tem que ser adicionado: _lv_group_add_obj()_
  • O grupo criado tem que se assinado para um dispositivo de entrada: _lv_indev_set_group(my_indev, group1);_
  • Use _LV_GROUP_KEY_..._ para navegar entre os objetos no grupo

Visite Touchpad-less navigation para aprender mais.

Codificador

Com um codificador voc pode fazer 4 coisas:

  1. pressionar seu boto
  2. pressionar longamente seu boto
  3. virar para esquerda
  4. virar para direita

Ao ligar o codificador voc pode focar no objeto prximo/anterior. Quando voc pressiona o codificador em um simples objeto (como um boto). ele ser clicado. Se voc pressiona o codificador sobre um objeto complexo (como uma lista, caixa de mensagem, etc.) o objeto ir para o modo edio onde ligar o codificador, voc pode navegar dentro do objeto. Para deixar o modo de edio pressione longamente o boto

indev_drv.type = LV_INDEV_TYPE_ENCODER;
indev_drv.read = my_input_read;

A funo de leitura deve se parecer como:

bool encoder_read(lv_indev_data_t*data){
  data->enc_diff = enc_get_new_moves();
  if(enc_pressed()){
    data->state = LV_INDEV_STATE_PR;
  }
  else{
    data->state = LV_INDEV_STATE_REL;
  }

  return false; /*Sem bufferizao ento nennhum dado para ler*/
}
  • Para usar um ENCODER, igualmente ao KEYPA, os objetos devem ser adicionados aos grupos.

Boto

indev_drv.type = LV_INDEV_TYPE_BUTTON;
indev_drv.read = my_input_read;

A funo de leitura deve se parecer como:

bool button_read(lv_indev_data_t*data){
    static uint32_t last_btn = 0;   /*Armazena o ltimo boto pressionado*/ 
    int btn_pr = my_btn_read();     /*Obtm o ID (0,1,2...) do boto pressionado*/
    if(btn_pr >= 0) {               /*Existe boto pressionado?*/  
       last_btn = btn_pr;           /*Salva o ID do boto pressionado*/
       data->state = LV_INDEV_STATE_PR;  /*Adiciona ao estado pressionado*/
    } else {
       data->state = LV_INDEV_STATE_REL; /*Adiciona ao estado liberado*/
    }

    data->btn = last_btn;            /*Adiciona o ltimo boto*/         

    return false;                    /*Sem bufferizao ento nennhum dado para ler*/
}
  • Os botes precisam ser assinados para pixels na tela usando lv_indev_set_button_points(indev, points_array). Onde _points_array_ se parece como const lv_point_t points_array[] = { {12,30},{60,90}, ...}

Interface tick

O LittlevGL usa um sistema tick. Chame a funo lv_tick_inc(tick_period) periodicamente e chame o perodo de chamada em milisegundos. Por exemplo, se voc chama em cada milisegundos: lv_tick_inc(1)

Carregamento de chamada

Para carregar as chamadas do LittlevGL voc precisa chamar lv_task_handler() periodicamente em uma das seguintes:

  • while(1) da funo main()
  • interrupo de temporizadores periodicos
  • uma tarefa do OS peridica

A temporizao no critica mas deve se em torno de 5 milisegundos para manter o sistema responsvel.

Exemplo:

while(1) {
  lv_task_handler();
  my_delay_ms(5);
}

O MCU pode ficar no estado sleep quando nenhuma entrada acontece. Nesse caso o main while(1) deve se parecer como:

while(1) {
  /*Operao normal em 1 seg*/ 
  if(lv_indev_get_inactive_time(NULL) < 1000) {    
    lv_task_handler();
  } 
  /*Dorme depois de 1 segundo inativo*/
  else {                                         
    timer_stop();               /*Para o temporizador quando lv_tick_inc()  chamado*/
    sleep();            /*Faz o MCU dormir*/
  }
  my_delay_ms(5);
}

Voc deve tambm adicionar essas linhas para seu dispositivo de entrara para ler a funo quando um pressionamento acontece:

/*Fora a tarefa de execuo quando acorda*/
lv_tick_inc(LV_REFR_PERIOD);
timer_start();               /*Reinicia o temporizador quando lv_tick_inc()  chamado*/
lv_task_handler();

Em adio lv_indev_get_inactive_time() voc pode checar lv_anim_count_running() para ver se cada animao foi finalizada.

Usando com um sistema operacional

LittlevGL not thread-safe. Alm disso, ele muito simples de usar dentro de um sistema operacional.

O cenrio simples no usar a tarefa do sistema operacional mas usar o lv_tasks. Um _lv_task_ uma fuuno chamada periodicamente em lv_task_handler. No _lv_task_ voc pode obter os estados dos sensores, buffers etc e chamar as funes do LittlevGL para atualizar a GUI. Para criar um _lv_task_ use: lv_task_create(my_func, period_ms, LV_TASK_PRIO_LOWEST/LOW/MID/HIGH/HIGHEST, custom_ptr)

Se voc precisa usar outras tarefas ou threads voc precisa de um mutex na qual deve ser tomado antes de ser chamado lv_task_handler e lanado depois dele. Em adio, voc ter que usar para esse mutex em outras tarefas e threads em volta de todo cdigo relacionado ao LittlevGL (lv_...). Desse jeito voc pode usar o LittlevGL em um ambiente real de multitarefa. Use um mutex para evitar chamadas concorrente das funes LittlevGL.

Instalando o SDL 2

O simulador PC usa a biblioteca de plataforma cruzada SDL 2 para simular um display TFT e o touch pad.

Linux

No Linux voc pode facilmente instalar o SDL2 usando um terminal:

  1. Encontre a verso atual do SDL2: apt-cache search libsdl2 (Ex.: libsdl2-2.0-0)
  2. Instale o SDL2: sudo apt-get install libsdl2-2.0-0 (troque com a verso encontrada)
  3. Instale o pacote de desenvolvimento SDL2: sudo apt-get install libsdl2-dev
  4. Se os pacotes essenciais no esto instalados ainda: sudo apt-get install build-essential
Windows

Se voc est usando o Windows primeiramente voc precisa instalar o MinGW (64 bit version). Depois disto siga os passoas para adicionar o SDL2:

  1. Faa download das bibliotecas do SDL. V para https://www.libsdl.org/download-2.0.php e faa download do Development Libraries: SDL2-devel-2.0.5-mingw.tar.gz
  2. Descompacte o arquivo e va para o diretrio _x86_64-w64-mingw32_ (para MinGW 64 bit) ou para i686-w64-mingw32 (para MinGW 32 bit)
  3. Copie a pasta _...mingw32/include/SDL2 para _C:/MinGW/.../x86_64-w64-mingw32/include_
  4. Copie o contedo _...mingw32/lib/ para _C:/MinGW/.../x86_64-w64-mingw32/lib_
  5. Copie _...mingw32/bin/SDL2.dll para _{eclipse_worksapce}/pc_simulator/Debug/_. Faa isso depois do Eclipse ser instalado.

Nota: Se voc usar o Microsoft Visual Studio ao invs do Eclipse ento voc no precisa instalar o MinGW

Mecanismo de trabalho dos objetos

Estrutura pai-filho

Um pai pode ser considerado como o container de seus filhos. Todos os objetos tem exatamente um objeto pai (exceto as telas), mas um pai pode ter ilimitado nmero de filhos. No existe nenhuma limitao para o tipo de pai, mas h tipicamente objetos pai (ex.: boto) e filho (ex.: rtulo).

Tela o pai mais bsico

A tela um objeto especial na qual no possui um objeto pai. Sempre existe uma tela ativa. Por padro, a biblioteca cria e carrega um. Para obter a tela ativa use a funo lv_scr_act().

Uma tela pode ser criada com qualquer tipo de objeto, por exemplo, um objeto bsico ou uma imagem para fazer um papel de parede.

Movendo junto

Se a posio do pai mudada, o filho ir mover junto com o pai. Contudo todas as posies so relativas ao pai. Para a coordenada (0;0), significa que os objetos permanecero no topo ao lado esquerdo do pai, independentemente da posio do pai.

lv_obj_t * obj1 = lv_obj_create(par, NULL);            /*Cria um objeto dentro do objeto pai criado anteriormente*/

Modificar a posio do pai

lv_obj_set_pos(par, 50, 50);        /*Move o pai. O filho ir se mover junto com ele.*/
Visibilidade somente no pai

Se um filho parcialmente ou totalmente est fora de seu pai ento as partes fora no ser visvel.

lv_obj_set_x(obj1, -30);        /*Move o filho um pequeno peda do pai*/

Cores

Escrito para v5.1

Os carregadores do mdulo de cor, todas as funes de cores relacionadas como mudar profundidade de cor, criando cor apartir do cdigo hexadecimal, converter profundidade de cor, mixando cores, etc.

Os tipos de variveis seguintes so definidos pelo modulo da cor:

  • lv_color1_t Guarda cor monocromtica. Para compatibilidade tambm contm campos R,G,B mas eles so sempre os mesmos (1 Byte)
  • lv_color8_t Uma estrutura para guardar componentes R (3 bit), G (3 bit), B (2 bit) para cores de 8 bits (1 Byte)
  • lv_color16_t Uma estrutura para guardar componentes R (5 bit), G (6 bit), B (5 bit) para cores de 16 bits (2 Bytes)
  • lv_color24_t Uma estrutura para guardar componentes R (8 bit), G (8 bit), B (8 bit) para cores de 24 bits (4 Bytes)
  • lv_color_t Igual para color 1/8/16/24_t de acordo para as configuraes de profundidade
  • lv_color_int_t uint8_t, uint16_t ou uint32_t de acordo para a configuraa de profundidade de cor. Usada para construir vetores de cor atravs de nmeros planos.
  • lv_opa_t Uma tipo simples uint8_t para descrever a opacidade.

Os tipos _lv_color_t_, _lv_color1_t__lv_color8_t_, _lv_color16_t_ e _lv_color24_t_ tem 4 campos:

  • red Canal vermelho
  • green Canal verde
  • blue Canal azul
  • full vermelho + verde + azul como um nmero

Voc pode configurar comprimento de cor atual em _lv_conf.h_ atualizando e definindo o _LV_COLOR_DEPTH_ para 1 (monocromtico), 8, 16 ou 24.

Voc pode converter uma cor de uma profundidade de cor atual para um outro. As funes conversoras retorna com um nmero, ento voc pode usar o campo full:

lv_color_t c;
c.red   = 0x38;
c.green = 0x70;
c.blue  = 0xCC;

c1.full = lv_color_to1(c);        /*Retorna 1 para cores de luzes, 0 para cores escuras*/

lv_color8_t c8;
c8.full = lv_color_to8(c);        /*D um nmero de 8 bit com a cor convertida*/ 

lv_color16_t c16;
c16.full = lv_color_to16(c);     /*D um nmero de 16 bit com a cor convertida*/

lv_color24_t c24;
c24.full = lv_color_to24(c);    /*D um nmero de 32 bit com a cor convertida*/

Voc pode criar uma cor com a profundidade de cor atual usando a macro LV_COLOR_MAKE. Ele exige 3 argumentos (vermelho, verde, azul) como nmero de 8 bit. Por exemplo, para criar uma cor vermelho claro: my_color = COLOR_MAKE(0xFF,0x80,0x80). Cores podem ser criadas de cdigo HEX tambm: my_color = LV_COLOR_HEX(0xFF8080) ou my_color = LV_COLOR_HEX3(0xF88).

Mixando duas cores possvel com mixed_color = lv_color_mix(color1, color2, ratio). A razo pode ser 0..255. 0 (zero) resulta completamente color2, 255 resulta completamente color1.

Fontes

Escrito para v5.1

As fontes LittlevGL so bitmaps e outros descritores para guardar as imagens das letras (glyph) e algumas informaes adicionais. Uma fonte armazenada em uma varivel lv_font_t e pode ser configurado no campo de estilo text.font.

As fontes tem uma propriedade bpp (Bit-Per-Pixel). Ele mostra como o bit usado para descrever um pixel na fonte. O valor armazenado para um pixel determina a opacidade do pixel. O jeito que a imagem das letras (especialmente sobre as bordas) podem ser suavizada. O valor bpp so 1, 2, 4 e 8 (maior valor significa melhor qualidade). O bpp tambm afeta o tamanho da memria requirida para armazenar a fonte. Ex.: bpp = 4 faz a memria da fonte 4 vezes maior do que bpp = 1.

Fontes embutidas

Existe muitas fontes embutidas na qual podem ser ativadas em lv_conf.h pelas definies _USE_LV_FONT_..._. Exinte fontes embutidas em estilos diferentes:

  • 10 px
  • 20 px
  • 30 px
  • 40 px

Voc pode ativar as fontes com valores 1, 2, 4 ou 8 para configurar seus bpp (ex.: #define USE_LV_FONT_DEJAVU_20 4 no lv_conf.h).

As fontes embutidas existem com multiplicar caracteres em cada tamanho:

  • ASCII (Unicode 32..126)
  • Suplemento Latim (Unicode 160..255)
  • Cirilico (Unicode 1024..1279)

As fontes embutidas usa a fonte Dejavu.

As fontes embutidas so variveis globais com nomes como:

  • lv_font_dejavu_20 (fonte ASCII de 20 px)
  • lv_font_dejavu_20_latin_sup (Fontes suplementar em Latin de 20 px)
  • lv_font_dejavu_20_cyrillic (Fonte cirlico de 20 px)

Suporte unicode

O LittlevGL suporta letras Unicode apartir da codificao de caracteres UTF-8. Voc precisa configurar seu editor para salvar seu cdigo/texto como UTF-8 (usualmente este o padro) e ativado _LV_TXT_UTF8_ no lv_conf.h. Sem a ativao do _LV_TXT_UTF8_ somente fontes ASCII e smbolos pode ser usado (veja os smbolos abaixo)

Depois disso os textos sero decodificados para determinar os valores Unicode. Para mostrar as letras sua fonte precisa conter a imagem (glyph) dos caracteres.

Voc pode assinar mais fontes para criar um conjunto de caracteres maior. Para fazer isso escolha a fonte base (tipicamente fonte ASCII) e adiciona a extenso a ele lv_font_add(child, parent). Somente fonte com mesmo tamanho sero assinadas.

As fontes embutidas j esto adicionadas ao mesmo tamanho da fonte ASCII. Por exemplo, se esto ativados os _USE_LV_FONT_DEJAVU_20_ e _USE_LV_FONT_DEJAVU_20_LATIN_SUP_ dentro do lv_conf.h ento o texto "abc" pode ser renderizado quando usar _lv_font_dejavu_20_.

Desenhos

Escrito para v5.1

No LittlevGL voc pode pensar em objetos grficos e no precisa se preocupar de como os desenhos acontecem. Voc pode configurar o tamanho, posio ou qualquer atributo do objeto e aa biblioteca ir atualizar as reas antigas (invlidas) e redesenhar as novas. Contudo, voc deve saber o bsico dos mtodos de desenho para saber em qual deva escolher.

Desenho buferizado e sem buffer

Desenho sem buffer

Os desenhos sem buffer coloca os pixels diretamente no display (frame buffer). Contudo durante o processo de desenho, alguma cintilao pode ser visvel porque primeiramente o background tem que ser desenhado e depois os objetos sobre ele. Por essa razo, este tipo no est disponvel quando deslizar, arrastar e animaes so usadas. Por outro lado, ele tem a menor memria porque nenhum buffer extra para grfico requerido

Para usar desenho no bufferizado configure _LV_VDB_SIZE_ para 0 dentro do _lv_conf.h_ e registre o _disp_map_ e _disp_fill_functions. Aqui voc pode aprender mais sobre Portabilidade.

Desenho bufferizado

O desenho bufferizado similar a bufferizao dupla quando duas telas com buffer dimensionado so usados (um para renderizao e outro para mostrar o ltimo quadro disponvel). Contudo, o algoritmo do desenho bufferizado do LittlevGL usa somente um frame buffer e um pequeno buffer chamado Virtual Display Buffer (VDB). Para o tamanho do VDB ~1/10 do tamanho da tela tipicamente o suficiente. Para uma tela de 320 x 240 com cores de 16-bit, necessrio somente 15 KB de RAM extra.

Com o desenho bufferizado no h cintilao porque a imagem criada primeiramente na memria (VDB), contudo deslizar, arrastas e animaes podem ser usadas. Em adio, ela tambm ativa o uso de outros efeitos grficos como anti-aliasing, transparencia (opacidade) e sombras.

Para usar desenho bufferizado configure _LV_VDB_SIZE_ para > LV_HOR_RES dentro do _lv_conf.h_ e registre uma funo _disp_flush_.

No modo bufferizado, voc pode usar VDB duplo para paralelamente executar renderizao dentro de um VDB e copiar pixels para seu frame buffer de um para outro. A cpia deve usar DMA ou outra acelerao de hardware para funcionar em background para deixar a CPU fazer outras tarefas. Dentro do _lv_conf.h_ o _LV_VDB_DOUBLE 1_ ativa esta caracterstica.

Desenho bufferizado vs desenho no bufferizado

Mantenha em mente que no garantido que o desenho no bufferizado mais rpido. Durante o processo de renderizao, um pixel sobrescrevido mltiplas vezes (ex.: backgroundm boto, texto esto acima um dos outros). Deste jeito em um modo no bufferizado a biblioteca precisa acessar a memria externa ou o controlador do display mltiplas vezes na qual mais lento do que escrever/ler a memria interna da RAM.

As tabelas seguintes resume as diferenas entre os dois mtodos de desenho:

Des. no buffer. Des. bufferizado
Uso da memria Sem extra >~1/10 da tela
Qualidade Cintilizao Sem falha
Antialiasing No suportado Suportado
Transparncia No suportado Suportado
Sombras No suportado Suportado

Anti-aliasing

Dentro do lv_conf.h voc pode ativar o anti-aliasing com _LV_ANTIALIAS 1_. O anti-aliasing suportado somente no mode bufferizado (LV_VDB_SIZE > LV_HOR_RES).

Os algoritmo anti-aliasing coloca alguns pixels transluzente (pixels com opacidade) para produzir linhas e curvas (incluindo bordas com raio) e at mesmo suavizamento. Porque ele coloca somente pixels anti-aliasing que requer somente um pouco de poder computacional extra (~1.1x do tempo extra comparado ao no configurao anti-aliasing)

Como descrito em Seo de fonte as fontes podem ser anti-aliased por usar uma fonte diferente com um bpp (Bit-Per-Pixel) maior. Deste jeito os pixels de uma fonte pode no somente ser 0 ou 1 mas ser transluzente. Os bpp-s suportados so 1, 2, 4 e 8. Mantenha na mente uma fonte com um maior bpp requer mais ROM.

Animaes

Escrito por v5.1

Voc pode automaticamente mudar o valor (animar) de uma varivel entre um incio e um valor fim usando funo animar com o prottipo `void func(void* var,int32_t value). A animao acontecer por uma chamada peridica da funo animadora com o parmetro correspondente.

Para criar uma animao voc tem que inicializar uma varivel _lv_anim_t_ (existe um template em lv_anim.h):

lv_anim_t a;                                
a.var = button1;                             /*Varivel para animar*/                        
a.start = 100;                                 /*Valor inicial*/
a.end = 300;                                 /*Valor final*/
a.fp = (lv_anim_fp_t)lv_obj_set_height;                                 /*Funo para ser usada para animar*/
a.path = lv_anim_path_linear;                                            /*Caminho da animao*/
a.end_cb = NULL;                               /*Callback quando a animao estiver pronta*/
a.act_time = 0;                                 /*Configura o tempo < 0 para um atraso [ms]*/
a.time = 200;                                 /*Comprimento da animao [ms]*/
a.playback = 0;                                 /*1: anima em direo reversa tambm quando o normal estiver pronto*/
a.playback_pause = 0;                                 /*Espera antes do playback [ms]*/
a.repeat = 0;                                 /*1: Repete a animao (com ou sem playback)*/
a.repeat_pause = 0;                             /*Espera antes de repetir [ms]*/

lv_anim_create(&a);                          /*Inicia a animao*/

A anim_create(&a) ir registrar a animao e imediatamente aplicar o valor start independentemente para configurar o atraso.

Voc pode determinar o atalho da animao. Na maioria simples do caso ele linear na qual significa os valores corrente start e end mudado linearmente. O atalho uma funo na qual calcula o prximo valor para configurar baseado no estado corrente da animao. Correntemente h dois atalhos embutidos:

  • lv_anim_path_linear animao linear
  • lv_anim_path_step muda em um passo no fim

Por padro, voc pode configurar o tempo da animao. Mas em alguns casos, a velocidade de animao mais prtica. A funo lv_anim_speed_to_time(speed, start, end) calcula o tempo requerido em milisegundos para alcanar o fim do valor apartir de um valor inicial com a dada velocidade. A velocidade interpretado em dimenso unit/sec. Por exemplo lv_anim_speed_to_time(20,0,100) nos dar 5000 milisegundos.

Voc pode aplicar animaes diferentes mltiplas na mesma varivel no mesmo tempo. (Por exemplo, animar as coordenadas x e y com _lv_obj_set_x_ e _lv_obj_set_y_). Mas somente uma animao pode existir com um uma dada varivel e um par de funes. Contudo a funo _lv_anim_create()_ apagar a funo existente de animao.

Voc pode apagar uma animao por lv_anim_del(var, func) com a varivel animada providenciada e sua funo animadora.

Matrix de botes (lv_btnm)

Escrito pora v5.1

Viso geral

Os objetos de Matrizes de Botes podem mostrar mltiplos botes de acordo para um descritor de vetor de string, chamada map. Voc pode especificar o mapa com lv_btnm_set_map(btnm, my_map).

A declarao de um mapa se parece como const char * map[] = {"btn1", "btn2", "btn3", ""}. Nota que o ltimo elemento tem que ser uma string vazia!

O primeiro caractere de uma string pode ser um controle de caractere para especificar alguns atributos:

  • bit 7..6 Sempre 0b10 para diferenciar o byte de controle vindo de caractere contextual
  • bit 5 Boto inativo
  • bit 4 Sem pressionamento longo para o boto
  • bit 3 Botes escondidos
  • bit 2..0 Comprimento relativo comparado para os botes na mesma coluna. [1..7]

recomendado para especificar o byte de controle como um nmero octal. Por exemplo "\213button". O nmero octal sempre comea com _2_ (bit 7..6) o meia parte os atributos (bit 5..3) e o ltima parte o comprimento (bit 2..0). Assim o exemplo descreve um boto escondido com 3 unidade de comprimento.

Use "\n" no mapa para fazer a quebra de linha: {"btn1", "btn2", "\n", "btn3", ""}. O comprimento do boto recalculado em cada linha.

A lv_btnm_set_action(btnm, btnm_action) especifica uma ao para chamar quando um boto liberado.

Voc pode ativar as comutao dos botes quando eles so clicados. Assim podem somente ser um boto comutado em uma vez. O lv_btnm_set_toggle(btnm, true, id) ativa a comutao e configura o _id_simo boto para o modo comutvel.

  3. Inicializar todos os pontos para um dado valor com: lv_chart_init_points(chart, ser, value).
  4. Configurar todos os pontos de um vetor com: lv_chart_set_points(chart, ser, value_array).

  • LV_CHART_TYPE_LINE: desenha linhas entre os pontos
  • LV_CHART_TYPE_COL: Desenha colunas
  • LV_CHART_TYPE_POINT: Desenha pontos

Voc pode especificar os tipos de display com lv_chart_set_type(chart, TYPE). O tipo LV_CHART_TYPE_LINE | LV_CHART_TYPE_POINT tambm vlido para desenhar ambas linhas e pontos.

Voc pode especificar os valores das direes min. e max. em y com lv_chart_set_range(chart, y_min, y_max). O valor dos pontos sero escaladas proporcionalmente. O alcance padro : 0..100.

O nmero de pontos na linhas de dados podem ser modificados por lv_chart_set_point_count(chart, point_num). O valor padro 10.

O nmero das linhas de divises horizontais e verticais podem ser modificados por lv_chart_set_div_line_count(chart, hdiv_num, vdiv_num). O padro de configuraes so linhas de diviso de 3 horizontais e 5 verticais.

Para configurar o comprimento de linha e ponto do raio use a funo lv_chart_set_series_width(chart, size). O padro : 2.

A *opacidade das linhas de dados podem ser especificadas pelo lv_chart_set_series_opa(chart, opa). O valor padro : OPA_COVER.

Voc pode aplicar um desaparecimento de cor escura sobre o fundo das colunas e pontos pela funo lv_chart_set_series_darking(chart, effect). O padro de nvel de escurido OPA_50.

Container (lv_cont)

Escrito para v5.1

Viso geral

Os containeres so objetos retangulares com algumas caractersticas especiais.

Voc pode aplicar um layout nos contineres para automaticamente ordenar seus filhos. O espaamento do layout vem das propriedades style.body.padding.hor/ver/inner. As possveis opes de layout:

  • LV_LAYOUT_OFF: No alinha as os filhos
  • LV_LAYOUT_CENTER: Alinha os filhos ao centro em colunas e mantm o espao pad.inner entre eles
  • LV_LAYOUT_COL_L: Alinha os filhos na coluna esquerda justificada. Mantm o espao pad.hor esquerda, espao pad.ver no topo e pad.inner entre os filhos.
  • LV_LAYOUT_COL_M: Alinha os filhos centralizados em coluna. Mantm o espao pad.ver no topo e o espao pad.inner entre os filhos.
  • LV_LAYOUT_COL_R: Alinha os filhos na coluna justificada direita. Mantm o espao pad.hor direita, o espao pad.ver no topo e o espao pad.inner entre os filhos.
  • LV_LAYOUT_ROW_T: Alinha os filhos na linha justificada ao topo. Mantm o espao pad.hor esquerda, o espao pad.ver ao topo e o espao pad.inner entre os filhos
  • LV_LAYOUT_ROW_M: Alinha os filhos em linhas centralizadas. Mantm o espao pad.hor esquerda e o espao pad.inner entre os filhos.
  • LV_LAYOUT_ROW_B: Alinha os filhos ao fundo da linha justificada. Mantm o espao pad.hor, o espao pad.ver ao fundo e o espao pad.inner entre os filhos.
  • LV_LAYOUT_PRETTY: Coloca muitos objetos possveis em uma linha (com pelo menos o espao pad.inner e o espao pad.hor ao lado) e comea uma nova linha. Divide o espao em cada linha igualmente entre os filhos. Mantm o espao pad.ver no topo e o espao pad.inner entre as linhas.
  • LV_LAYOUT_GRID: Similar ao PRETTY LAYOUT mas no divide o espao horizontal igualmente apenas deixa o espao pad.hor

Voc pode ativar um mtodo auto ajuste na qual automaticamente configura o tamanho do continer para incluir todos os filhos. Ele manter o espao pad.hor esquerda e direita e o espao pad.ver no topo. O auto ajuste pode ativar horizontalmente, verticalmente o em ambas as direes com a funo lv_cont_set_fit(cont, true, true). O segundo parmetro a horizontal, o terceiro parmetro a ativao do ajuste vertical.

Teclado (lv_kb)

Escrito para v5.1

Viso geral

Como o nome mostra o objeto Teclado provm um teclado para escrever texto. Voc pode assinar uma rea de texto para o Teclado para colocar os caracteres digitados l. Para assinar a rea de texto use lv_kb_set_ta(kb, ta).

O Teclado contm um boto _Ok_ e um boto Hide. Em aes ok e hide voc podem ser especificados por lv_kb_set_ok_action(kb, action) e lv_kb_set_hide_action(kb, action) para adicionar callbacks para os cliques Ok/Hide. Se nenhuma ao especificada ento os botes ir apagar o Teclado.

A rea de texto assinada cursor pode ser gerenciado teclado: quando o teclado assinado a rea de texto prvio do cursor ser escondido e a nova rea de texto ser mostrada. Clicando em _Ok_ ou Hide ir tambm esconder o cursor. O gerenciador do cursor ativado pelo lv_kb_set_cursor_manage(kb, true). O padro no gerenciar.

Os teclados tem dois modos:

  • LV_KB_MODE_TEXT: mostra letras, nmeros e caracteres especiais
  • LV_KB_MODE_NUM: mostra nmeros, sinais +/- e pontos

Para configurar o modo use lv_kb_set_mode(kb, mode). O padro _LV_KB_MODE_TEXT_

Voc pode especificar um novo mapa (layout) para o teclado com lv_kb_set_map(kb, map). Ele funciona como a Matriz de botes assim, controlar caractere pode ser adicionado ao layout para configurar o comprimento do boto e outros atributos. Mantenha em mente que usar as seguintes palavras-chave ter o mesmo efeito com o mapeamento original: _SYMBOL_OK_, _SYMBOL_CLOSE_, _SYMBOL_LEFT_, _SYMBOL_RIGHT_, ABC, abc, Enter, Del, _#1_, +/- .

Uso do estilo

O teclado funciona com 6 estilos: um background e 5 estilos de botes para cada estado. Voc pode confiigurar estilos com lv_kb_set_style(btn, LV_KB_STYLE_..., &style). O background e os botes use a propriedade style.body. Para os rtulos use a propriedade style.text dos estilos de botes.

  • LV_KB_STYLE_BTN_REL estilo ao deixar de pressionar. Padro: _lv_style_btn_rel_
  • LV_KB_STYLE_BTN_PR estilo ao pressionar. Padro: _lv_style_btn_pr_
  • LV_KB_STYLE_BTN_TGL_REL estilo ao comutar. Padro: _lv_style_btn_tgl_rel_
  • LV_KB_STYLE_BTN_TGL_PR estilo do boto comutado. Padro: _lv_style_btn_tgl_pr_
  • LV_KB_STYLE_BTN_INA estilo do boto inativo. Padro: _lv_style_btn_ina_

Escrito para v5.1

Os rtulos so objetos bsicos para mostrar texto. No existe limitao no tamanho do texto porque ele guardado dinamicamente. Voc pode modificar o texto em tempo real a qualquer momento com lv_label_set_text().

Voc pode usar \n para fazer a quebra de linha. Por exemplo: "line1\nline2\n\nline4"

  • LV_LABEL_LONG_BREAK: Mantm o comprimento do objeto, quebra as linhas muito longas e expande a altura do objeto
  • LV_LABEL_LONG_DOTS: Mantm o tamanho do objeto, quebra o texto e escreve pontos na ltima linha
  • LV_LABEL_LONG_SCROLL: Expande o tamanho do objeto e desliza o texto no pai (move o objeto rtulo)
  • LV_LABEL_LONG_ROLL: Mantm o tamanho e o rolamento somente o texto (no o objeto)

Voc pode especificar o modo longo com: lv_label_set_long_mode(label, long_mode)

Rtulos so capazes de mostrar texto de um vetor esttico. Use lv_label_set_static_text(label, char_array). Neste caso, o texto no guardado na memria dinmica mas o vetor dado usado. Mantenha em mente que o vvetor no pode ser uma varivel local na qual se destroi quando a funo deixa de ser executada.

O rtulo do texto pode ser alinhado esquerda, direita ou centro com lv_label_set_align(label, LV_LABEL_ALIGN_LEFT/RIGHT/CENTER)

Voc pode ativar o desenhar um background para o rtulo com lv_label_set_body_draw(label, draw)

No texto, voc pode usar comandos para recolorizar partes do texto. "Write a #ff0000 red# word". Por exemplo: Neste recurso pode ser ativado individualmente para cada rtulo pela funo lv_label_set_recolor()

Os estilos padres dos rtulos NULL assim eles herdam o estilo do pai.

Pgina (lv_page)

Escrito para v5.1

Viso geral

A Pgina consiste de dois continer em cada outro: o boto o background (ou base) e o top o deslizvel. Se voc criou um filho na pgina ele ser automaticamente movido em um continer deslizvel. Se o continer deslizvel se tornar maior do que o background ele pode ser deslizvel pelo arrasto (como a lista de smart phones).

Por padro o atributo auto fit do atributo deslizvel ativado verticalmente assim sua altura ser aumentada para incluir seus filhos. O comprimento do deslizvel automaticamente ajustado pelo comprimento do background (menos o espaamento horizontal do background).

O objeto background pode ser referenciado como a pgina dele mesmo como: lv_obj_set_width(page, 100).

O objeto de rolagem pode ser recuperada com: lv_page_get_scrl(page).

Barra de rolagem pode ser mostrado de acordo a quatro polticas:

  • LV_SB_MODE_OFF: Nunca mostra barra de rolagem
  • LV_SB_MODE_ON: Sempre mostra barra de rolagem
  • LV_SB_MODE_DRAG: Mostra barra de rolagens quando a pagina est sendo arrastada
  • LV_SB_MODE_AUTO: Mostra barra de rolagem quando o continer deslizvel longo o bastante para ser deslizado

Voc pode configurar a barra de rolagem mostranto poltica pelo: lv_page_set_sb_mode(page, SB_MODE). O valor padro _LV_PAGE_SB_MODE_ON_;

Voc pode colar um filho para a pgina. Neste caso voc pode deslizar a pgina com arrasto do objeto filho. Ele pode ser ativado pelo lv_page_glue_obj(child, true).

Voc pode focar um objeto sobre uma pgina com: lv_page_focus(page, child, anim_time).

Ele mover o continer deslizvel para mostrar um filho. Se o ltimo parmetro no zero ento a pgina se mover com uma animao.

Uma ao ao soltar e ao pressionar podem ser assinadas para a Pgina com lv_page_set_rel_action(page, my_rel_action) e lv_page_set_pr_action(page, my_pr_action). A ao pode ser gatihada de um Background e um objeto deslizvel tambm.

Existem funes para direcionar atribuir/obter atributos dos deslizveis:

  • lv_page_set_scrl_fit()
  • lv_page_set_scrl_width()
  • lv_page_set_scrl_height()
  • lv_page_set_scrl_layout()

Uso do estilo

Use lv_page_set_style(page, LV_PAGE_STYLE_..., &style) para atribuir um novo estilo para um elemento da pgina:

  • LV_PAGE_STYLE_SCRL estilo deslizvel na qual usa todas as propriedades style.body (padro: lv_style_pretty)
  • LV_PAGE_STYLE_SB estilo da barra deslizvel na qual usa todas as propriedades style.body. Espaamentos horizontal/vertical configura o espaamento da barra de rolagem respectivamente a o espaamento interno configura o comprimento da barra de rolagem (padro: lv_style_pretty_color)

Mas uma parte deslizvel no pode estar no meio assim ele ser movido de volta ao topo. Para evitar isso use lv_obj_align() para trocar o objeto relatiovo a cada outro (um tem que estar no topo/esquerda) ou desativado o ajuste com lv_page_set_scrl_fit(page, false, false); e configure seu tamanho lv_page_set_scrl_width/height(page, 100).
  • O background desenha sua borda quando o deslizvel desenhado. Ele garante que a pgina sempre ter que ser uma forma fechada at mesmo se um deslizvel tem a mesma cor da pgina pai

  • LV_SW_STYLE_INDIC Estilo do indicador. Todas as propriedades style.body so usadas. Os valores de padding fazem o indicador menor do que o background.
  • LV_SW_STYLE_KNOB_OFF Estilo da ala quando ela estiver desligada. A propriedade style.body usada exceto o espaamento.
  • LV_SW_STYLE_KNOB_ON Estilo da ala quando a chave est ativada. A propriedade style.body usada exceto o espaamento.

  • A ala no um objeto real ele somente um desenho acima da barra

void callback(lv_obj_t * tabview, uint16_t act_id);

rea de texto (lv_ta)

Escrito para v5.1

Viso geral

A rea de Texto uma pgina com um rtulo e um cursor nele. Voc pode inserir texto ou caracteres para a posio atual com:

  • lv_ta_add_char(ta, 'c')
  • lv_ta_add_text(ta, "insert this text")

O lv_ta_set_text(ta, "New text") muda o texto todo.

Para apagar um caractere da esquerda da posio do cursor use lv_ta_del_char(ta).

A posio do cursor pode ser modificada diretamente como lv_ta_set_cursor_pos(ta, 10) ou por passo:

  • lv_ta_cursor_right(ta)
  • lv_ta_cursor_left(ta)
  • lv_ta_cursor_up(ta)
  • lv_ta_cursor_down(ta)

Existem vrios tipos de cursores. Voc pode configurar um deles com: lv_ta_set_cursor_type(ta, LV_CURSOR_...)

  • LV_CURSOR_NONE
  • LV_CURSOR_LINE
  • LV_CURSOR_BLOCK
  • LV_CURSOR_OUTLINE
  • LV_CURSOR_UNDERLINE

Voc pode 'OR' _LV_CURSOR_HIDDEN_ para qualquer tipo para esconder o cursor.

A rea de texto pode ser configurada para ser uma linha com lv_ta_set_one_line(ta, true).

A rea de texto suporta modo de senha. Ele pode ser ativado com lv_ta_set_pwd_mode(ta, true).

Uso do estilo

Use lv_ta_set_style(page, LV_TA_STYLE_..., &style) para configurar um novo estilo para um elemento da rea de texto:

  • LV_TA_STYLE_BG estilo do background na qual usa todas as propriedades style.body. O rtulo tambm usa esse style.label deste estilo. (padro: lv_style_pretty)
  • LV_TA_STYLE_SB estilo de barra de rolagem na qual usa todas as propriedades do style.body (padro: lv_style_transp)

  • LV_TA_STYLE_CURSOR estilo de cursor. Se NULL ento a biblioteca configura para ns um estilo automaticamente de acordo com a cor do rtulo e fonte

    • LV_CURSOR_LINE: uma longa style.line.width linha mais desenhado como um retangulo como espaamento style.body horizontais e verticais d um espao sobre o cursor
    • LV_CURSOR_BLOCK: um retangulo como espaamentos style.body horizontais e verticais que fazem o retangulo mais largo
    • LV_CURSOR_OUTLINE: um retangulo vazio (somente uma borda) como espaamentos style.body horizontais e verticais que fazem o retangulo mais largo
    • LV_CURSOR_UNDERLINE: uma linha de comprimento style.line.width mas desenhada como retangulo como espaamentos horizontais e verticais que fazem um offset sobre o cursor

Notas

  • No modo senha lv_ta_get_text(ta) te d o texto real e no o caractere asteristico

Janela (lv_window)

Escrito para v5.1

Viso geral

As janelas so um dos mais complexos continer similares a objetos. Eles so construdos de duas partes principais: um cabealho Continer no topo e uma Pgina para o contedo abaixo do cabealho.

Sobre o cabealho h um ttulo na qual pode ser modificado por: lv_win_set_title(win, "New title"). O ttulo sempre herda o estilo do cabealho.

Voc pode adicionar botes de controle para o lado direito do cabealho com: lv_win_add_btn(win, "U:/close", my_close_action). O segundo parmetro um atalho de arquivo de imagem, o terceiro parmetro uma funo para chamar quando o boto solto. Voc pode usar smbolos como imagens tambm como: lv_win_add_btn(win, SYMBOL_CLOSE, my_close_action).

Voc pode modificar o tamanho do controle de botes com a funo lv_win_set_btn_size(win, new_size).

O comportamento da barra de rolagem pode ser configurado pelo lv_win_set_sb_mode(win, LV_SB_MODE_...).

Para confugurar um layout para o contedo use lv_win_set_layout(win, LV_LAYOUT_...).

Uso do estilo

Use lv_win_set_style(win, LV_WIN_STYLE_..., &style) para configurar um novo estilo para um elemento da janela:

  • LV_WIN_STYE_BG background principal na qual usa todas as propriedades de style.body (cabealho e pgina de contedo so colocadas nele) (padro: lv_style_plain)
  • LV_WIN_STYLE_CONTENT_BG contedo do background da pgina na qual usa todas as propriedades de style.body (padro: lv_style_transp)
  • LV_WIN_STYLE_CONTENT_SCRL contedo da pgina da parte deslizvel na qual usa todas as propriedades de style.body (padro lv_style_transp)
  • LV_WIN_STYLE_SB estilo da barra de deslizamento na qual usa todas as propriedades de style.body. Espaamentos hor/ver* configuram o espaamento da barra de rolagem respectivamente e seus espaamentos internos configuram o comprimento da barra de rolagem. (padro: lv_style_pretty_color)
  • LV_WIN_STYLE_HEADER estilo do cabealho na qual usa todas as propriedades de style.body (padro: lv_style_plain_color)
  • LV_WIN_STYLE_BTN_REL estilo do boto solto (sobre o cabealho) na qual usa todas as propriedades de style.body (padro: lv_style_btn_rel)
  • LV_WIN_STYLE_BTN_PR estilo do boto solto (sobre o cabealho) na qual usa todas as propriedades de style.body (padro: lv_style_btn_pr)
